## 1️⃣ לולאת הפניות (Referral Loop) — הפיצ'ר המרכזי

### למה זה חשוב
שלושת התחומים נבחרו בדיוק בגלל שהם מלווים את אותה עסקת נדל"ן ומפנים לקוחות זה לזה.
היום ההפניה קיימת רק כמקור ליד פסיבי ("הפניית סוכן נדל\"ן"). הפיצ'ר הופך אותה לפעולה
בתוך המערכת — וכל הפניה לקולגה שעוד לא רשום היא גם **מנוע צמיחה ויראלי**: כל משתמש
מגייס את שני בעלי המקצוע שסביב העסקאות שלו.

### חוויית משתמש
1. בכרטיס ליד (בדרך כלל אחרי "עסקה נסגרה"): כפתור **"🔗 הפנה לקולגה"**.
2. בוחרים תחום יעד (עו"ד נדל"ן / מעצב פנים / סוכן נדל"ן) + שם וטלפון של הקולגה.
3. המערכת יוצרת קישור הפניה חד-פעמי ופותחת וואטסאפ עם הודעה מוכנה:
   *"היי עו"ד כהן, סגרתי עסקה עם משפחת לוי על דירה ברעננה — הם צריכים ליווי משפטי.
   כל הפרטים כאן: {link}"*.
4. הקולגה פותח את הקישור:
   - **רשום ב-Liders** → מסך "התקבלה הפניה" + כפתור "קבל את הליד" → הליד נוצר אצלו
     עם מקור "הפניית {תחום}" וקרדיט למפנה.
   - **לא רשום** → דף נחיתה ייעודי: "קיבלת הפניית לקוח מ{שם המפנה}" + הרשמה מהירה
     (30 יום חינם). אחרי האונבורדינג הליד כבר מחכה בפייפליין. **Aha moment מיידי.**
5. המפנה רואה במסך ייעודי: כמה הפניות שלח, כמה התקבלו, כמה הפכו לעסקאות.
6. גיימיפיקציה: ‎+250 XP על הפניה שהתקבלה, ‎+500 XP על הפניה שהפכה לעסקה סגורה,
   הישג "מלך ההפניות" (10 הפניות שהתקבלו).

### מבנה נתונים (מיגרציה חדשה)
```sql
CREATE TABLE referrals (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  from_tenant_id uuid NOT NULL REFERENCES tenants(id),
  from_user_id uuid NOT NULL,
  lead_id uuid REFERENCES leads(id) ON DELETE SET NULL,
  lead_snapshot jsonb NOT NULL,          -- שדות מסוננים בלבד: שם, טלפון, אזור, הקשר
  to_vertical text NOT NULL CHECK (to_vertical IN ('realestate','realestate_lawyer','interior')),
  to_name text, to_phone text,
  token text UNIQUE NOT NULL,            -- אקראי, חד-פעמי
  status text NOT NULL DEFAULT 'sent'    -- sent → opened → accepted → converted / expired
    CHECK (status IN ('sent','opened','accepted','converted','expired')),
  accepted_by_tenant_id uuid REFERENCES tenants(id),
  created_at timestamptz DEFAULT now(),
  expires_at timestamptz DEFAULT now() + interval '14 days'
);
-- RLS: המפנה רואה את ההפניות שלו; המקבל רואה הפניות שהתקבלו אצלו. אין anon SELECT.
```

### RPCs
| פונקציה | תפקיד |
|---------|-------|
| `create_referral(lead_id, to_vertical, to_name, to_phone)` | יוצר snapshot מסונן + token; rate limit ‎10/יום/משתמש |
| `get_referral_preview(token)` | **anon** — מחזיר תצוגה מוגבלת (שם מפנה, תחום, שם ליד בלבד) לדף הנחיתה; מסמן `opened` |
| `accept_referral(token)` | authenticated — יוצר ליד אצל המקבל, מסמן `accepted`, מזכה את המפנה ב-XP |
| `list_my_referrals()` | סטטיסטיקות למסך ההפניות של המפנה |

### אבטחה (חובה לפני מימוש — `/security-review`)
- ה-snapshot **לא** כולל הערות פנימיות, ערך עסקה או היסטוריית פעילות — רק שם, טלפון,
  אזור ומשפט הקשר שהמפנה כותב.
- token אקראי (32 bytes), תפוגה 14 יום, שימוש חד-פעמי. `get_referral_preview` עם
  rate limit לפי IP (למניעת סריקת tokens).
- אישור ההפניה דורש הרשמה/כניסה — אין גישת anon לנתוני הליד המלאים.
- חוק הגנת הפרטיות: המפנה מאשר checkbox "קיבלתי את הסכמת הלקוח להעברת פרטיו".

### שלבי מימוש
1. **MVP**: מיגרציה + כפתור בכרטיס ליד + קישור וואטסאפ + דף קבלה למשתמש רשום.
   ✅ **בוצע 3/7/2026** — מיגרציה `061_lead_referral_loop.sql` + `LeadReferral` ב-index.html.
2. **צמיחה**: דף נחיתה לקולגה לא-רשום + חיבור לאונבורדינג.
   ✅ **בוצע חלקית 3/7/2026** — הטוקן (`?lref=`) נשמר ב-localStorage ושורד הרשמה/
   אונבורדינג; מודל הקבלה מופיע אחרי ההתחברות הראשונה. נותר: דף נחיתה ייעודי עם
   פרטי המפנה לפני ההרשמה.
3. **ליטוש**: מסך סטטיסטיקות, XP והישגים, התראת Make למפנה כשההפניה מתקבלת.
   ⏳ בוצע רק ‎+250 XP על הפניה שהתקבלה (`pull_lead_referral_xp`).

## 2️⃣ התאמת נכס ↔ לידים (סוכן נדל"ן)

### הרעיון
כשסוכן מכניס נכס חדש, ה-AI סורק את הלידים הפתוחים ומציע: *"3 לידים מחפשים בדיוק
את זה"* + טיוטת הודעת וואטסאפ אישית לכל אחד.

### מימוש
- **שלב א' (בלי סכמה חדשה)**: כפתור "🎯 מצא לידים מתאימים" בכרטיס ליד מסוג נכס /
  במסך שיווק. ה-AI (דרך `ai-proxy`, מכסת `general`) מקבל את תיאור הנכס + רשימת
  לידים פתוחים (שם, תקציב, אזור, הערות) ומחזיר עד 3 התאמות מנומקות + טיוטת הודעה.
- **שלב ב'**: שדות מובנים ללידים של סוכן נדל"ן (תקציב, אזור מבוקש, חדרים) כדי לשפר
  דיוק — jsonb `lead_prefs` על `leads`, בלי מיגרציה כבדה.
- עלות AI: קריאת general רגילה (~$0.0012) — בתוך המכסות הקיימות, אפס שינוי תמחור.

## 3️⃣ ציר זמן תיק ללקוח (עו"ד נדל"ן)

### הרעיון
עו"ד מגדיר לכל תיק שלבים (קבלת תיק → בדיקת נאותות → טיוטת חוזה → חתימה → דיווח
מיסוי → רישום), והלקוח מקבל **קישור לצפייה בלבד** שמראה איפה התיק עומד. חוסך את
שיחות "מה קורה עם התיק שלי?" — הכאב מספר 1 של לקוחות עו"ד.

### מימוש
- עמודת `case_timeline jsonb` על `leads` (מערך שלבים: שם, סטטוס, תאריך). ברירת
  מחדל: תבנית 6 השלבים; עו"ד יכול לערוך.
- UI בכרטיס ליד (מוצג רק כש-`industry='realestate_lawyer'`): checklist שלבים
  עם סימון "הושלם".
- שיתוף ללקוח: שימוש חוזר בתשתית ה-share הקיימת (shared_leads) עם view מצומצם —
  **רק** שמות שלבים, סטטוס ותאריכים. בלי סכומים, בלי הערות, בלי פרטי צדדים אחרים.
- זה גם אבן הדרך הראשונה לקראת פיצ'ר **החתימה הדיגיטלית** שברודמאפ — אותו מסך
  לקוח יארח בהמשך את "מסמך ממתין לחתימתך".

## 4️⃣ סיכום תיק השראה ב-AI (מעצב פנים)

### הרעיון
שדה ההשראה (Pinterest) כבר קיים אצל מעצבי פנים. כפתור חדש: **"✨ סכם קונספט"** —
ה-AI הופך את קישורי/תיאורי ההשראה + הערות הליד לתוצר מקצועי:
סיכום סגנון (2-3 משפטים) + פלטת צבעים מוצעת + 3 כיווני קונספט + פסקה מוכנה
להצעת מחיר.

### מימוש
- קריאת `ai-proxy` עם מכסת `marketing` (זה תוכן שיווקי/מכירתי) — חסום מאחורי
  `_requireAccess()` הקיים של תוסף השיווק. **ערך מוסף לתוסף ה-₪100/חודש.**
- בלי סכמה חדשה: הקלט הוא שדה ההשראה + הערות; הפלט נשמר כהערה בליד.
- הערה: ה-AI לא גולש לקישורי Pinterest (אין fetch) — הוא עובד על הטקסט/התיאורים.
  בעתיד אפשר לשקול צירוף תמונות דרך Claude vision.
