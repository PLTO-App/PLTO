# סשן הבא: התאמות per-vertical לעו"ד נדל"ן ומעצב פנים

## מה כבר מוכן (בוצע בסשן רון לנדסמן)
- ✅ PropertyShown — מוסתר מעו"ד ומעצב פנים (מוצג רק ל-`realestate`)
- ✅ `AI.writeRescue()` — מותאם לכל תחום (עדכון שוק / עדכון משפטי / טרנד עיצובי)
- ✅ תבנית WA `cold_referral` — מותאמת לכל תחום (לקנות/למכור → ייעוץ משפטי / פרויקט עיצוב)
- ✅ `commission_renewal_date/notes` — קיים בטבלה (מיגרציה 076), מוצג לכולם בשם "עמלה"

---

## משימות לסשן הבא

### 1. תווית "עמלה" דינמית לפי תחום
**איפה:** modal עריכת ליד (`modal-edit-lead`) + info tab (`LeadDetail.render()`)

| תחום | label נוכחי | label רצוי |
|------|-------------|------------|
| סוכן נדל"ן | תאריך חידוש עמלה | תאריך חידוש עמלה |
| עו"ד נדל"ן | תאריך חידוש עמלה | תאריך חידוש שכ"ט |
| מעצב פנים | תאריך חידוש עמלה | תאריך סיום פרויקט |

**שינוי קל:** להוסיף helper function:
```javascript
function renewalLabel() {
  const ind = localStorage.getItem('plto_industry') || State.tenant?.industry || 'other';
  return ind === 'realestate_lawyer' ? 'תאריך חידוש שכ"ט'
       : ind === 'interior'          ? 'תאריך סיום פרויקט'
       : 'תאריך חידוש עמלה';
}
```

ולהחליף 3 מקומות:
- label במודל עריכת ליד (`id="edit-lead-renewal-date"`)
- badge בinfo tab (`📋 ${label}`)
- כותרת widget בדשבורד (`📋 חידוש עמלה — בקרוב`)

### 2. placeholder שדה "אזור" לפי תחום
**איפה:** modal הוספת ליד + modal עריכה — שדה `desired_area`

| תחום | placeholder נוכחי | placeholder רצוי |
|------|-------------------|-----------------|
| סוכן נדל"ן | "תל אביב, גוש דן..." | ללא שינוי |
| עו"ד נדל"ן | "תל אביב, גוש דן..." | "אזור בו מצוי הנכס/הפרויקט" |
| מעצב פנים | "תל אביב, גוש דן..." | "עיר / שכונה של הפרויקט" |

---

### 3. כותרת "נכסים תואמים" בtab-match לפי תחום
**איפה:** `LeadDetail.render()` — tab match, שורת הטקסט `נמצאו X נכסים תואמים`

| תחום | טקסט נוכחי | טקסט רצוי |
|------|------------|-----------|
| סוכן נדל"ן | נמצאו X נכסים תואמים לפי תקציב ואזור | ללא שינוי |
| עו"ד נדל"ן | נמצאו X נכסים תואמים לפי תקציב ואזור | נמצאו X נכסים/תיקים רלוונטיים |
| מעצב פנים | נמצאו X נכסים תואמים לפי תקציב ואזור | נמצאו X פרויקטים תואמים לתקציב |

---

### 4. empty state בtab-match לפי תחום
**איפה:** `LeadDetail.render()` — `empty-title` בtab match

| תחום | כרגע | רצוי |
|------|------|------|
| סוכן נדל"ן | לא נמצאו נכסים תואמים | ללא שינוי |
| עו"ד נדל"ן | לא נמצאו נכסים תואמים | לא נמצאו נכסים/תיקים רלוונטיים |
| מעצב פנים | לא נמצאו נכסים תואמים | לא נמצאו פרויקטים תואמים |

---

### 5. כפתור tab "התאמת נכסים" לפי תחום
**איפה:** `getLabel('matchTab')` — כנראה כבר מוגדר ב-`getLabel()`, לבדוק

```javascript
// לבדוק בקוד:
getLabel('matchTab')
// אם מחזיר "נכסים" — להתאים לפי תחום
```

---

### 6. AI writeFollowup — לא דחוף, אבל שווה לבדוק
**איפה:** `AI.writeFollowup()` — prompt כולל "אזור" ו"תקציב" בלי הסיאק

כרגע:
```
אזור: ${lead.desired_area || 'לא ידוע'} | תקציב: ${lead.budget_max ? ...}
```
לעו"ד: "אזור" = מיקום הנכס/הפרויקט — בסדר
למעצב: "אזור" = מיקום הפרויקט — בסדר
הטקסט כנראה בסדר כי הוא כללי מספיק. לא דחוף.

## הערות

- כל השינויים לעיל הם **טקסטואליים בלבד** — אין מיגרציות DB, אין לוגיקה חדשה
- PropertyShown לעו"ד ולמעצב: **לא רלוונטי** (עו"ד לא מציג נכסים, מעצב מציג "השראה" לא נכסים)
  - בסשן הבא אפשר לשקול: הוספת "מעקב פרויקטים שהוצגו" **ייעודי** למעצב פנים
    (כתובת → סגנון → תגובה: אוהב/לא בטוח/לא מתאים — בסשן עתידי נפרד)
- לנדינג — סקשן "הכאבים שסוכן נדל"ן מכיר" כרגע מציג רק כאבי סוכן. בסשן הבא
  אפשר להוסיף סקשן מקביל לעו"ד ולמעצב — אם יש מסלולי שיווק נפרדים.
